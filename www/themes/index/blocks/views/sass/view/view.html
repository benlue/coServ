如何使用

在coServ，SCSS依照用途放在不同路徑

  1. blocks
  2. cont/shared/sass

blocks/views/[myBlock]

網頁切板常以區塊區分，這也是coServ載入頁面的最小單位，而一個block 會有這些檔案
  • blocks
    • views
      • myBlock
        • myBlock.html
        • myBlock.css
        • myBlock.js
        • myBlock.lang /* 多國語言設定檔 */
        • myBlock.scss /* Here! */

當你希望scss只在某區塊生效，試試放在這

cont/shared/sass

希望讓myBlock.scss引用scss code，而不用重複放在各個Blocks？

試試放這，適合整理mixin library或自定的mixin.scss

整理 code

  • /* cont/shared/sass/example.scss */
    • .
    • .
    • ....[your mixin]
  • /* myBlock.scss */
    • @import "example.scss"; /* 載入自定mixin */
    • .
    • .
    • ....[your scss code]

如何運作

載入順序

你可能會有疑問，有CSS又有SCSS，那coServ會怎麼處理呢？

先載入Blocks內的.css，找不到才載入.scss

會和css互衝？

原生SASS編譯後，會產生.css檔

但coServ直接幫你在網頁加上scss編譯完成的css code，不會產生.css檔。

自行擴充

Mixin Library 有

Compass
Bourbon
...等選擇，選一套自己喜歡的

為例

  1. npm install node-bourbon --save
  2. find "drink bourbon", 移除註解
  3. /* myBlock.scss */
    4. @import "bourbon"; /* 載入bourbon mixin*/
    • ....
    • .
    • .
    • ....[your scss code]

這樣就能使用Bourbon的mixin了！
